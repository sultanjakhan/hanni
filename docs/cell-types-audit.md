# Аудит типов ячеек таблиц

Каждый тип описан: рендер, редактор, проблемы, идеи.
Статус: ❌ требует работы | ⚠️ работает, но есть минусы | ✅ ОК

---

## 1. text (Текст) ✅
- **Рендер:** `escapeHtml(val)` — просто текст
- **Редактор:** `<input type="text">`, blur/tab/enter сохраняют
- **Проблемы:** нет поддержки многострочного текста (textarea)
- **Идеи:** добавить тип `long_text` для заметок/описаний

## 2. number (Число) ✅
- **Рендер:** `escapeHtml(val)` — просто число как текст
- **Редактор:** `<input type="number" step="any">`
- **Валидация:** `!isNaN(parseFloat(v))`
- **Проблемы:** нет форматирования (тысячи, валюта, десятичные)
- **Идеи:** опциональный формат (₸, $, %, пробелы в тысячах)

## 3. select (Выбор) ⚠️
- **Рендер:** `<span class="badge badge-{color}">value</span>`, цвет по индексу из 8 цветов
- **Редактор:** dropdown с поиском, создание новых опций, переименование, удаление
- **Проблемы:**
  - Цвет привязан к позиции в массиве, а не к опции — переименование/удаление ломает цвета
  - Нет возможности выбрать цвет для конкретной опции
  - fixedColumns хранят каталог опций в localStorage (отдельно от БД)
  - customProperties хранят опции в БД (через `update_property_definition`)
  - Два разных пути = потенциальная рассинхронизация
- **Идеи:**
  - Хранить цвет вместе с опцией `{value, label, color}`
  - Ручной выбор цвета для каждой опции (как в Notion)

## 4. multi_select (Мульти-выбор) ⚠️
- **Рендер:** массив badges через `JSON.parse(val)`
- **Редактор:** multi-dropdown с чекбоксами, поиск, создание, удаление
- **Проблемы:**
  - Те же проблемы с цветами что у select
  - Значение хранится как JSON-строка `["a","b"]` — хрупко
  - В ячейке badges могут не влезть — нет wrap/overflow
- **Идеи:**
  - `+2 ещё` если не влезает
  - Цвет привязать к опции

## 5. date (Дата) ✅
- **Рендер:** `d.toLocaleDateString('ru-RU')` в `<span class="cell-date">`
- **Редактор:** `<input type="date">`, сохранение на `change`
- **Проблемы:**
  - Нативный date picker — не стилизуется, выглядит чужеродно
  - Нет поддержки диапазонов дат
- **Идеи:** свой date picker с подсветкой "сегодня", "завтра"

## 6. time (Время) ✅
- **Рендер:** `<span class="cell-time">HH:MM</span>`, tabular-nums
- **Редактор:** два input'а [ЧЧ] : [ММ], автопереход, валидация 0-23/0-59
- **Проблемы:** минимальные

## 7. checkbox (Чекбокс) ✅
- **Рендер:** `<span class="cell-check-round checked/empty">` с SVG галочкой
- **Редактор:** toggle по клику (true ↔ false), без popup
- **Проблемы:** минимальные
- **Идеи:** —

## 8. url (Ссылка) ⚠️
- **Рендер:** `<a href="..." target="_blank" class="cell-link">`, обрезка до 30 символов
- **Редактор:** `<input type="text">` — обычный текстовый ввод
- **Проблемы:**
  - Клик по ссылке открывает URL вместо редактирования
  - Нет favicon или preview
  - Обрезка жёстко 30 символов, не адаптивная
- **Идеи:**
  - Иконка ↗ для открытия, клик по ячейке = редактирование
  - Показывать домен вместо полного URL

## 9. email (Email) ⚠️
- **Рендер:** `<a href="mailto:..." class="cell-link">`
- **Редактор:** `<input type="text">`
- **Проблемы:**
  - Клик открывает почтовый клиент вместо редактирования
  - Нет валидации email формата
- **Идеи:** валидация, иконка для mailto

## 10. phone (Телефон) ⚠️
- **Рендер:** `<a href="tel:..." class="cell-link">`
- **Редактор:** `<input type="tel">`
- **Проблемы:**
  - Клик открывает FaceTime/звонок вместо редактирования
  - Нет маски ввода (+7 XXX XXX XX XX)
- **Идеи:** маска телефона, форматирование

## 11. progress (Прогресс) ✅
- **Рендер:** прогресс-бар (max 80px) + число `N%`
- **Редактор:** range slider 0-100, live display
- **Проблемы:**
  - Работает только для customProperties, не для fixedColumns
  - fixedColumns с progress используют editType:'number' (обычный input)
- **Идеи:** поддержка в fixedColumns

## 12. rating (Рейтинг) ✅
- **Рендер:** 5 звёзд ★, filled/empty, жёлтый цвет
- **Редактор:** 5 интерактивных звёзд, клик = установить, повторный = сбросить
- **Проблемы:**
  - Только для customProperties, fixedColumns используют editType:'number'
  - Нет hover preview при наведении на звёзды в режиме отображения
- **Идеи:** поддержка в fixedColumns

## Общие проблемы

1. **Цвета select/multi_select** — привязаны к индексу позиции, а не к опции. При удалении/переименовании цвета "плывут"
2. **Ссылочные типы (url/email/phone)** — клик открывает ссылку, а не редактор. Конфликт click = navigate vs click = edit
3. **fixedColumns vs customProperties** — разная логика хранения опций (localStorage vs БД)
4. **progress/rating в fixedColumns** — используют fallback на number input вместо своих специализированных редакторов
