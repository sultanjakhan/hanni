# B1 — System Prompt: Изменения v0.18.0

> A. System Prompt Refinement

## Что изменилось

### SYSTEM_PROMPT (lib.rs:18)

Добавлен блок `RESPONSE QUALITY` (6 правил) и `EXAMPLES OF GOOD RESPONSES` (2 примера).

**Новые правила:**
1. **Step-by-step** — для сложных вопросов: думать поэтапно внутри
2. **Эмпатия** — сначала отреагировать на эмоции/состояние
3. **Уточнение** — задать ОДИН follow-up при неоднозначном запросе
4. **Адаптивная длина** — простой = 1-2 предложения, сложный = 3-6
5. **Без вербатим-повтора** — не повторять сообщение пользователя дословно
6. **Натуральная память** — "Ты же вроде учишься в KBTU..." вместо "Согласно моей памяти..."

**Примеры Good/Bad:**
- Эмоциональный ввод ("устал") → Good: эмпатия + предложение из памяти
- Вопрос с инструментом ("сколько потратил") → Good: вызов инструмента + саммари

### SYSTEM_PROMPT_LITE (lib.rs:33)

+3 правила:
- Эмоция-первая: если пользователь делится эмоциями — отреагировать на них
- Вариация: иногда вопрос, иногда комментарий, иногда юмор
- Натуральные ссылки на память, когда релевантно

### is_complex_query() (lib.rs:74)

Новая функция-классификатор:
```rust
fn is_complex_query(user_msg: &str) -> bool
```

| Условие | Порог |
|---------|-------|
| Длина сообщения | > 100 символов |
| Маркеры-слова | 13 слов: "почему", "как лучше", "объясни", "сравни", "что думаешь", "помоги выбрать", "расскажи подробн", "в чём разница", "что лучше", "как правильно", "посоветуй", "проанализируй", "зачем" |

Используется в:
- **A**: complex hint в system_content (chat_inner)
- **D**: условие срабатывания self-critique
