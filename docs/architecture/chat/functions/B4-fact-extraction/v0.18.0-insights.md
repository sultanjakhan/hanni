## Новая таблица — init_db (lib.rs:1269)

```sql
CREATE TABLE IF NOT EXISTS conversation_insights (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    conversation_id INTEGER NOT NULL,
    insight_type    TEXT NOT NULL,     -- 'decision' | 'open_question' | 'topic' | 'action_taken'
    content         TEXT NOT NULL,     -- текст инсайта на русском
    created_at      TEXT NOT NULL      -- ISO 8601
);
```

Нет FK на conversations — облегчает запись, insights полезны даже если conversation удалён.

## Расширение промпта — process_conversation_end (lib.rs:4695)

**Было:**
```
{"summary": "...", "facts": [...]}
```

**Стало:**
```
{"summary": "...", "facts": [...], "insights": [{"type": "...", "content": "..."}]}
```

Добавлена секция `INSIGHTS — Types:` в промпт:

| Тип | Описание | Пример |
|-----|---------|--------|
| decision | Выбор пользователя | "решил перейти на утренние тренировки" |
| open_question | Нерешённый вопрос/дилемма | "выбирает между React и Vue" |
| topic | Основная тема обсуждения | "обсуждали карьеру" |
| action_taken | Действия в ходе разговора | "добавил задачу на завтра" |

`max_tokens`: 800 → 1000 (для дополнительного JSON-блока)

## Новые структуры — lib.rs:4796

```rust
#[derive(Deserialize)]
struct ExtractedInsight {
    #[serde(rename = "type")]
    insight_type: String,
    content: String,
}
```

Добавлено в `ExtractionResult`:
```rust
#[serde(default)]
insights: Vec<ExtractedInsight>,
```

`#[serde(default)]` — обратная совместимость: если LLM не вернёт insights → пустой вектор.

## Сохранение — lib.rs:5040

После сохранения фактов (dedup pipeline):
```rust
for insight in &result.insights {
    let itype = insight.insight_type.as_str();
    if matches!(itype, "decision" | "open_question" | "topic" | "action_taken") {
        conn.execute(
            "INSERT INTO conversation_insights (conversation_id, insight_type, content, created_at)
             VALUES (?1, ?2, ?3, ?4)",
            params![conversation_id, itype, insight.content, now],
        );
    }
}
```

Фильтр `matches!()` отсекает невалидные типы.

## Инъекция в chat_inner — lib.rs:3815

Добавлено к существующему блоку `[Recent conversations]`:

```sql
SELECT insight_type, content, created_at
FROM conversation_insights
WHERE insight_type IN ('decision', 'open_question')
ORDER BY created_at DESC
LIMIT 8
```

Формат в контексте:
```
[Recent decisions & open questions]
- [decision] 2026-02-22: решил перейти на утренние тренировки
- [open_question] 2026-02-21: выбирает между React и Vue
```

Только `decision` и `open_question` — `topic` и `action_taken` слишком шумные для контекста.

Условия инъекции (как у `[Recent conversations]`):
- `messages.len() <= 4` (начало разговора)
- `!call_mode`
