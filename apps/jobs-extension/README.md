# Hanni Job Tracker — Chrome-расширение

Кнопка поверх страницы вакансии: отметить отклик (этап, контакт, заметка) и
сохранить прямо в таблицу **Jobs** в Hanni. Работает на любом сайте —
парсер берёт данные из JSON-LD (`JobPosting`), знает hh.kz/hh.ru, LinkedIn,
Habr Career, Djinni, GeekJob, а на незнакомых страницах поля можно
поправить руками.

## Установка в Chrome

1. Открой `chrome://extensions`
2. Включи **Developer mode** (переключатель справа сверху)
3. Нажми **Load unpacked** и выбери папку `apps/jobs-extension`
4. Кликни иконку расширения → вставь API-токен:
   ```sh
   cat ~/Library/Application\ Support/Hanni/api_token.txt | pbcopy
   ```
5. Порт: **8235** — обычный Hanni, **8236** — dev-сборка (`cargo tauri dev`)
6. «Проверить связь» — должно показать «Связь и токен в порядке ✓»
   (Hanni при этом должен быть запущен)

## Как пользоваться

- На странице вакансии справа внизу появляется фиолетовая кнопка **H**
  (если вакансия распозналась автоматически).
- Клик → панель с предзаполненными полями: позиция, компания, зарплата,
  этап (по умолчанию «Отклик»), контакт, источник, заметка.
- Если вакансия уже есть в базе (по URL) — панель покажет текущий этап,
  сохранение **обновит** запись, а не создаст дубль.
- На странице без кнопки: иконка расширения → «Отметить вакансию на этой
  странице» — панель откроется принудительно.

## Как это устроено

- `parser.js` — вытаскивает поля со страницы (JSON-LD → селекторы сайта →
  og:title/h1 fallback)
- `content.js` — кнопка + панель, общение с background
- `background.js` — service worker, ходит в Hanni API
  (`GET/POST http://127.0.0.1:<port>/api/vacancy`, Bearer-токен)
- Rust-сторона: `desktop/src-tauri/src/api_jobs.rs` — lookup по URL и
  upsert в таблицу `job_vacancies` (этап `applied` проставляет `applied_at`)
