# Hanni Job Tracker — Chrome-расширение

Постоянная панель сбоку браузера (Chrome Side Panel): показывает вакансию
с активной вкладки, отмечает отклик (этап, контакт, заметка) и сохраняет
прямо в таблицу **Jobs** в Hanni. Работает на любом сайте — парсер берёт
данные из JSON-LD (`JobPosting`), знает hh.kz/hh.ru, LinkedIn, Habr Career,
Djinni, GeekJob, а на незнакомых страницах поля можно поправить руками.

## Установка в Chrome

1. Открой `chrome://extensions`
2. Включи **Developer mode** (переключатель справа сверху)
3. Нажми **Load unpacked** и выбери папку `apps/jobs-extension`
4. Токен подхватывается **автоматически** из `token.local.js` (gitignored,
   уже создан). В настройках панели будет зелёное «Токен из token.local.js ✓».
   Если токен Hanni пересоздан (ротация) — перегенерируй файл одной командой
   и перезагрузи расширение:
   ```sh
   cd apps/jobs-extension && printf "self.HANNI_LOCAL_TOKEN = '%s';\n" \
     "$(cat ~/Library/Application\ Support/Hanni/api_token.txt)" > token.local.js
   ```
   Fallback без файла: вставить токен руками в настройках панели — из
   вставленного автоматически вычленяется UUID (лишний мусор отбрасывается).
5. Порт (чипы): **8235** — обычный Hanni, **8236** — dev-сборка (`cargo tauri dev`).
   Если на выбранном порту нет роута вакансий (старая сборка) или сервер не
   отвечает — расширение само пробует второй порт.

## Как пользоваться

- Клик по иконке расширения → **панель открывается сбоку и висит постоянно**,
  пока сам её не закроешь. При переключении вкладок и переходах по страницам
  она сама перечитывает вакансию (кнопка ↻ — перечитать вручную).
- Поля предзаполнены парсером: позиция, компания, зарплата, этап
  (по умолчанию «Отклик»), контакт, источник, заметка.
- Если вакансия уже есть в базе (по URL) — панель покажет текущий этап,
  сохранение **обновит** запись, а не создаст дубль.

### Авто-отклик на hh.kz / hh.ru

- Откликнулся на вакансию (на её странице) → расширение само ставит этап
  **«Отклик»** в таблице (и создаёт запись, если её не было). Тост
  «Hanni: отклик отмечен ✓» подтверждает.
- Открыл **«Мои отклики»** на hh → статусы синкаются пачкой:
  «Отказ» → Отказ, «Приглашение» → Интервью, «(не) просмотрено» → Отклик.
- Только вверх по воронке: авто-синк никогда не откатывает этап назад
  (интервью не станет «откликом», отказ не затрётся).

Быстрые альтернативы без side panel (панелька внутри страницы):
- Кнопка **H** справа внизу любой страницы — открыть/закрыть
- **Правый клик** → «Отметить вакансию в Hanni»
- **Хоткей** ⌥H / Alt+H (поменять: `chrome://extensions/shortcuts`)

## Как это устроено

- `parser.js` — вытаскивает поля со страницы (JSON-LD → селекторы сайта →
  og:title/h1 fallback)
- `sidepanel.html` / `sidepanel.js` — side panel: форма, настройки,
  слежение за активной вкладкой
- `content.js` — in-page кнопка + панель, отдаёт парсинг side panel'у
- `background.js` — service worker, ходит в Hanni API
  (`GET/POST http://127.0.0.1:<port>/api/vacancy`, Bearer-токен; фоллбэк
  порта при 404/недоступности)
- `token.local.js` — токен из `api_token.txt` (генерится командой выше,
  gitignored); имеет приоритет над введённым вручную
- Rust-сторона: `desktop/src-tauri/src/api_jobs.rs` — lookup по URL и
  upsert в таблицу `job_vacancies` (этап `applied` проставляет `applied_at`)
